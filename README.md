## Introduction
[cite_start]**MoodMate** is a high-fidelity interactive UI/UX prototype developed as part of the **ISB 36403 - Interaction Design** course at **Universiti Kuala Lumpur (UniKL)** during the October 2024 session[cite: 2136, 2138, 2139, 2141]. [cite_start]The project demonstrates the implementation of user-centered design principles through a practical mental health and wellness management application[cite: 2203, 2417].

[cite_start]The goal of this project is to provide a hands-on application of interaction design operations — empathizing with users, prototyping, and usability testing — while delivering a user-friendly graphical experience built with Figma[cite: 2294, 2297, 2640, 2656].

## Problem Statements & Objectives

### Problem Statements
* [cite_start]**Restrictive & Inaccessible Design**: Existing mental health apps limit utility for users who cannot afford premium subscriptions and exclude visually impaired users with poor contrast[cite: 2236, 2249].
* [cite_start]**Confusing Interfaces & Lack of Guidance**: Users often fail to interpret their progress effectively due to confusing graphs and feel overwhelmed by insufficient onboarding[cite: 2261, 2263, 2267, 2269].
* [cite_start]**Lack of Action Control**: Users experience frustration when unable to reverse or modify inputs, such as undoing actions submitted to an AI[cite: 2272, 2275].

### Objectives
* [cite_start]Implement a fully functional **user-centered design** to manage a mental wellness app[cite: 2294].
* [cite_start]Apply UI/UX principles including **Shneiderman's Eight Golden Rules**, **Feedback**, **Visibility**, and **Affordance**[cite: 2533, 2552, 2555, 2558].
* [cite_start]Build an interactive **Figma-based high-fidelity prototype** with seamless navigation[cite: 2418, 2640].
* [cite_start]Evaluate usability using the **System Usability Scale (SUS)** to gather actionable user feedback[cite: 2656, 2657].

## Program Scope
The MoodMate Prototype allows users to:
* [cite_start]**Add and Track Moods** through daily check-ins using emoji-based selections[cite: 2436, 2459].
* [cite_start]**Review Health Data** with categorized reports and vitals monitoring (heart rate, weight, blood group)[cite: 2468, 2469].
* [cite_start]**Explore Content** with personalized recommendations, trending videos, and goal-based browsing filters[cite: 2458, 2460, 2488, 2489].
* [cite_start]**Communicate** with mental health professionals in real-time via the secure SafeSpace chat interface[cite: 2525, 2528, 2529].

[cite_start]The application is prototyped in **Figma**[cite: 2640]. [cite_start]Internally, the app is structured using clear information architecture including authentication, health dashboards, and settings[cite: 2422].

## Information Architecture
[cite_start]The prototype is implemented using a categorized card-based structure and a persistent bottom navigation bar to manage different user flows[cite: 2423, 2431].

`[Authentication] -> [Home Dashboard] <-> [My Health] <-> [Discover] <-> [SafeSpace] <-> [Profile]`

### Module Structure
* [cite_start]**Authentication**: Entry point for manual and social logins[cite: 2447, 2448].
* [cite_start]**Home Page**: Core hub for personalized recommendations and check-ins[cite: 2458, 2459].
* [cite_start]**My Health**: Detailed reports and vitals tracking[cite: 2468, 2469].
* [cite_start]**SafeSpace**: Real-time professional support messaging[cite: 2528].

| Module | Responsibility |
| :--- | :--- |
| `Authentication` | [cite_start]Handles manual sign-up, social logins, and password recovery [cite: 2447, 2448, 2449] |
| `Home / Discover` | [cite_start]Displays personalized content, quick access cards, and interactive browsing [cite: 2458, 2488] |
| `My Health` | [cite_start]Manages categorized health reports, vitals tracking, and educational articles [cite: 2468, 2469, 2477] |
| `SafeSpace` | [cite_start]Provides a secure interface for real-time text-based chat with experts [cite: 2437, 2528, 2529] |
